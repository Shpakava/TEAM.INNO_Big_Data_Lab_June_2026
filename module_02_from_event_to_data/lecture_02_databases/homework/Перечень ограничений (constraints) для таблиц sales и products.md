Ограничения NOT NULL (Запрет пустых значений)

Для таблицы products (Товары):
product_name: не может быть NULL. 
• Обоснование: товар без названия не имеет смысла для покупателя и сотрудников. Нельзя отобразить его на витрине или напечатать в чеке.
price: не может быть NULL.
• Обоснование: в коммерческом магазине каждый товар должен иметь определенную стоимость. СУБД не сможет рассчитать сумму продажи, если цена будет отсутствовать.
category_id: не может быть NULL.
• Обоснование: каждый продукт должен принадлежать к определенной категории для правильного начисления налогов, инвентаризации и отчетности.

Для таблицы sales (Продажи):
product_id: не может быть NULL.
• Обоснование: нельзя зафиксировать факт продажи, если неизвестно, какой именно товар был продан.
quantity: не может быть NULL.
• Обоснование: система должна точно знать, какое количество товара было приобретено, чтобы корректно списать остатки со склада.
sales_timestamp: не может быть NULL.
• Обоснование: время совершения финансовой операции критически важно для налоговой отчетности, бухгалтерского учета и анализа продаж.


Ограничения уникальности.
product_id (в таблице products) должен быть уникальным.
• Обоснование: это первичный ключ (PRIMARY KEY), который однозначно идентифицирует каждый конкретный товар в каталоге.
sales_id (в таблице sales): должен быть уникальным.
• Обоснование: это первичный ключ для каждой строки продажи, гарантирующий, что каждая запись о покупке является единственной в своем роде.
product_name (в таблице products): желательно сделать уникальным.
• Обоснование: предотвращает случайное создание двух абсолютно одинаковых товаров, что привело бы к путанице на складе.

Ограничения проверки (CHECK constraints)
Проверка цены товара (таблица products):
• Цена товара обязательно должна быть больше нуля. Магазин не может продавать товары бесплатно или с отрицательной стоимостью.
Проверка количества проданного (таблица sales):
• Количество продаваемого товара в чеке должно быть строго положительным числом. Нельзя продать 0 товаров или отрицательное количество.
Проверка размера скидки (таблица sales):
• Если скидка выражается в процентах, она не может быть отрицательной и не может превышать 100%.
Проверка срока годности (таблица products):
• Срок годности продуктов питания должен составлять как минимум 1 день. Значение не может быть нулевым или отрицательным.
